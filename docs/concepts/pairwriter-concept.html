System Architecture — PairWriter

A thinking workspace with memory: an environment where reasoning and record-keeping coexist and reinforce each other, keeping the document permanently linked to the context that created it.

AI changed 1 passage — Section 3 · Context Engine
Accept
Reject

1. Core Flow — Think → Structure → Document

The core workflow differs radically from current tools. ChatGPT, Notion AI, and similar editors assume the user already knows what they want to write. PairWriter solves the problem that comes before that: helping professionals discover what they think before they document it.

  • The document remains present but can stay blank while the thinking matures in chat
  • Critical points are marked as checkpoints — decisions that must not be lost in the context window
  • When the thinking is mature, the document is generated or updated inline, without leaving context
  • The AI knows the full history and can evolve the document precisely at any future moment

2. Document Model as the Central Unit

Each document is a self-contained entity: it carries multiple chat sessions with complete history, linked checkpoints, version history, and references to related documents. The central metaphor is RAW photography — the document is the exportable RAW file and the chat history is the .xmp sidecar containing the full reasoning.

Multiple sessions solve the context-window problem: when the chat becomes too long, the user starts a new session focused on a specific aspect without losing the previous reasoning, which is preserved as a distilled checkpoint in SQLite.

3. AI Context Engine

This is the most critical section of the product. The engine determines what the AI receives on each call — and therefore the quality of the entire experience.

Context composition follows a strict priority order: system prompt → active checkpoints → current document → session history → files enabled by the knowledge base toggle. When the token budget is exceeded, the truncation strategy preserves checkpoints and discards the oldest chat history first.

C# · ContextBuilder.cs Copy 
// Build context payload respecting token budget
var ctx = new ContextBuilder(maxTokens: model.ContextWindow)
  .Add(systemPrompt,   priority: Priority.Critical)
  .Add(checkpoints,   priority: Priority.High)
  .Add(activeDocument, priority: Priority.High)
  .Add(sessionHistory, priority: Priority.Medium, truncatable: true)
  .Add(knowledgeBase,  priority: Priority.Low,    truncatable: true)
  .Build();

4. Arquitetura de Dados — Dual SQLite

O produto usa dois bancos com responsabilidades distintas. O .pairwriter.sqlite lives inside each project folder — portable, travels with the folder — and stores chat, checkpoints, versions, and relationships between documents. The global.sqlite vive em ~/.pairwriter/ and stores reusable patterns and a cross-project personal library.

Git is not a dependency. O arquivo .md on disk is only the readable mirror — the export. Each document has an immutable internal UUID generated at creation. Divergence detection from external edits is handled through a content hash.
